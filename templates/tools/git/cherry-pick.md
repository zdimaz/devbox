# Git Cherry-pick

## 🧠 Суть

Перенести один конкретный коммит из одной ветки в другую — без мержа всего остального.

**Когда нужен:**
- Хотфикс из `feature` нужен в `master` прямо сейчас, не дожидаясь PR
- Коммит случайно попал не в ту ветку
- Нужно портировать фикс из `preprod` обратно в `master`

---

## 💻 Базовое использование

```bash
# Найти hash нужного коммита
git log --oneline feature/some-branch

# Перенести один коммит в текущую ветку
git cherry-pick a1b2c3
```

---

## 💻 Несколько коммитов

```bash
# Диапазон коммитов (a1b2c3 не включается, f4e5d6 включается)
git cherry-pick a1b2c3..f4e5d6

# Вразброс
git cherry-pick a1b2c3 d4e5f6 g7h8i9
```

---

## 💻 Перенос без автокоммита

Посмотреть изменения перед фиксацией:

```bash
git cherry-pick a1b2c3 --no-commit
git status       # проверить что попало в индекс
git diff --staged
git commit
```

---

## 💻 Конфликт при cherry-pick

```bash
# Разрешить конфликт вручную в файлах, затем:
git add .
git cherry-pick --continue

# Отменить — вернуться к состоянию до cherry-pick:
git cherry-pick --abort
```

---

## ⚠️ Подводные камни

- Cherry-pick создаёт **новый коммит** с тем же содержимым, но другим hash — оригинал в исходной ветке остаётся
- Если тот же коммит позже попадёт через merge — возможны дубли изменений и конфликты
- Не злоупотребляй: если нужно перенести много коммитов — лучше merge или rebase
